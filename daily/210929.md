어떤 피드백들이 있을까?   
- 테스트 코드 
    - 구현하는 어플리케이션 중 가장 중요한 로직을 테스트 코드 작성한다
- 코드리뷰 
    - 3주전에 내가 짠 코드 후에 본다. 50분 코드짜고, 10분 셀프 코드리뷰
- 짝 프로그래밍 
    - 강추! 
    - 두명이서 진행하기 때문에 터널 비젼에 들어갈 일이 없음
    - 사간을 나눠서 드라이버와 네비게이터 을 정하고 정해진 시간만큼 진행 & 서로 역할을 바꿈
- 커뮤니티 참여
    - 코테 풀고, 커뮤니티에 올려서 다른 방법있는지 질문 함
    - 내가 시간을 3~4배를 사용하는 걸 효과적으로 다른 해결 방법을 알아낼 수 있다
    - 알고리즘을 푼 다음에 이걸 해결할 수 있는 다릉 방법은 무엇인지 알아본다    
	-> 다각도에서 문제를 해결할 수 있는 역랑이 생김
- 화면 녹화 등
    - 내 커서가 많이 머물러 있는지

# Part3> What 그래서 오늘 해야하는 이것

되돌아 보다[회고]   
어렵지 않음. 숨쉬듯이 할 수 있다.

어떤 회고가 좋은 회고인가?

```
5f
- fact - 무슨 일이 있었는지, 뭘 했는지
- feeling - 무엇을 느꼈는지
- finding - 어떤 인사이트가 있었는지
- future action - 액션 플랜이 있는지
- feedback - 진행한 액션에 플랜에 본인의 피드백
```

자기전에 일기 쓰기 -> 한꺼번에 하루라는 양을 기록하니 지속적으로 하기 어려움   
하루를 짧게 나눠서 하면 좋음   



```
// ex)

5f
- fact - 이 영상을 보고 어던 걸 알게 되었고
- feeling - 어떤 걸 느꼈으며
- finding - 내가 몰랐던 어떤 인사이트를 엳었고
- future action - 당장 무엇을 할 수 있는지
- feedback - 진행한 액션에 언제 피드백 체크할 것인지
```
