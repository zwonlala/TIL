## CH1. 정규표현식 개요

정규표현식(Regular Expression)은 문자열을 **대상** 으로 **조건** 을 지정하여 검색/치환/검사를 **실행**

즉,

- 대상
- 조건
- 실행

이 정규표현식의 3대 요소

대상은 이미 문자열로 정해졌으니   
-> '조건과 실행을 배우는 것' 이 정규표현식을 배우는 것

특히 \<실행> 에는 검색/치환/검사를 행하는 메소드를 의미하고   

6개가 존재

- '검색'은 결과를 반환하고
- '치환'은 값을 바꾸고
- '검사'는 true/false를 반환

-----

&nbsp;

정규표현식도 클래스이므로    
new 연산자를 사용하여 인스턴스를 생성한 후    
메소드를 호출해야 하지만 인스턴스를 생성하지 않아도 됨   

-> JS는 인스턴스 생성하지 않고도 메소드를 호출할 수 있는 메커니즘을 제공   
-> _(...? 무슨 개념..?)_


&nbsp;


- new 연산자
- RegExp() : 정규표현식 생성자 함수

를 같이 사용하여 인스턴스 생성할 수 있음   
-> 재사용할 수 있다는 장점이 있음!

-----

&nbsp;


검색 기준 : '정규 표현식 패턴' or '패턴'

패턴은 슬래스 두개 `/ /`로 표기

슬래시 사이에 기준을 작성함

슬래시 사이에 작성한 기준을 통칭하여 패턴이라고 함

&nbsp;

### '치환' replacement

대상 문자열에 패턴을 적용하여 추출할 값을 별도로 지정한 값으로 바꾸는 것

일반적으로 변경할 값으로 패턴 형태가 아닌 '문자열로' 지정함

&nbsp;


### '매치' match

패턴이 결정되면 대상 문자열에 패턴을 적용하게 되는데 이를 매치라고 함

패턴을 기준으로 매치 대상을 검색/치환/검사를 실행하는 건 메소드가 결정함


패턴은 크게
- 문자열 값으로 매치
- 패턴 문자를 지정하여 패턴으로 매치

하는 2가지 형태가 존재

--------

&nbsp;

### 관련 메소드

JS 의 함수는 단독으로 실행할 수 있음   
(오브젝트를 지정하지 않으면 window 오브젝트에 설정되고, window 오브젝트를 지정하지 않아도 호출이 됨)


하지만 메소드는 오브젝트가 있어야 함.

정규표현식에서 사용하느 메소드는 모두 오브젝트가 있으며 함수 형태가 아님!

but, 정규표현식에서 사용하는 메소드는 오브젝트를 지정하지 않음!!! 

(∵ JS가 메소드 앞에 작성한 데이터 타입을 분석하여 String 클래스 또는 RegExp 클래스에 속한 메소드를 호출하기 때문!)

∴ object_name 위치에 데이터를 지정해야 함


&nbsp;

#### String 클래스
- `match`
- `search`
- `replace`
- `split`

#### RegExp 클래스
- `exec`
- `test`

### 정규표현식 작성법

1. 대상 문자열을 메소드 앞에 작성하고 메소드의 파라미터에 패턴을 지정하는 방법
    - String 클래스에 정의된 메소드 사용하는 경우
    - 장) 직관적으로 코드를 이해 가능
    - 단) 패턴이 길어지면 가독성이 좋지 않아지며, 재사용이 불가
        => 변수에 패턴을 할당한 다음 재사용하는 구조가 좋음
2. 패턴을 메소드 앞에 작성하고 메소드의 파라미터에 대상 문자열을 지정하는 방법
    - RegExp 클래스에 정의된 메소드 사용하는 경우
    - 여기서도 패턴 부분을 변수에 할당한 다음 재사용하는 구조로 사용하는게 좋음
    - 패턴을 정의할때 실수로 \'\' 와 같은 따옴표로 패턴을 묶으면 안됨!
3. RegExp 인스턴스를 생성하고 메소드를 호출하는 방법
    - `new RegExp(PATTERN [, FLAG])`
        - PATTERN : 'sp'와 같이 문자열로 패턴을 지정
            - Ex/ `vat regExp1 = new RegExp('sp', 'i');`
        - FLAG : i, g, n 중에 해당하는 플래그
        - new 연산자를 사용하지 않고 생략해도 new 연산자를 사용한 것 처럼 RegExp 인스턴스가 반환됨
        - 인스턴스 생성과 메소드 호출을 연결해서 작성할 수 있으나 인스턴스를 재활용 못하므로 비추
