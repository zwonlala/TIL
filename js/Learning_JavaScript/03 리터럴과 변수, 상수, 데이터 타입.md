3.6 숫자

숫자의 근사치를 저장
-> IEEE-764 배정도 double-precision 부동소수점 숫자 형식(double-precision)
∴ 근사치기 때문에 이상한 현상이 있음

Ex) JS에서 0.1 + 0.2는 0.3 이 아닌 0.30000000000000004 임
(안믿긴다면, 크롬의 개발자 도구 콘솔에서 '0.1 + 0.2'를 치면 위 결과 나옴)


JS 에서는 4가지 숫자형 리터럴을 인식

- 10진수 리터럴

	- 소수점 있는 10진수
	- 지수표기법
- 2진수
- 8진수
- 16 진수	

추가적으로

- 무한대 ∞
- 음의 무한대 -∞
- '숫자가 아님' NaN

const inf = Infinity;
const ninf = -Infinity;
const nan = NaN; //숫자가 아님

숫자가 어떤 리터럴 형식으로 사용되더라도 결국 숫자는 더블 형식으로 저장됨.

3.12 객체
원시타입은 단 하나의 값만 나타낼 수 있고 불변, 
But 깩체는 여러가지 값이나 복잡한 값을 나타낼 수 있고 변할 수 도 있음


객체의 본질은 컨테이너!!!
(컨테이너의 내용물은 시간이 지나면서 바뀔 수 O,
but 내용물이 바뀐다고 컨테이너가 바뀐것이 아님!
여전히 같은 객체임!)

객체에는 {} (중괄호)를 사용하는 리터럴 문법이 있음

객체의 콘텐츠는 프로퍼티 property 혹은 멤버 member라고 불림
프로퍼티는 이름(키)과 값으로 구성

프포퍼티의 이름은 반드시 문자열이나 심볼잉어야 하고, 값은 상관없음


obj.color = 'yellow';

프로퍼티 이름에 유효한 식별자를 써야 
멤버 접근 연산자(member access operator .)를 사용할 수 있음


obj["not an identifier"] = 3;
obj["not an identifier"]; //3
obj["color"]; 					//"yellow"

프로퍼티 이름에 유효한 식별자가 아닌 이름을 쓴다면 계산된 멤버 접근 연산자를(computed member access operator []) 써야 함

프로퍼티 이름이 유효한 식별자라도 대괄호로 접근할 수 있음!


심볼 프로퍼티에 접근할 때도 대괄호를 사용함

const SIZE = Symbol();
obj[SIZE] = 8;
obj[SIZE]; 			//8


obj에는 
- 유효한 식별자 문자열 ("color")
- 유효한 식별자가 아닌 문자열 ("not an identifier")
- 심볼 (SIZE)
이렇게 3가지 프로퍼티가 있음

여기서 유효한 식별자가 아닌 문자열이 공백 때문인지?

Note) JS 콘솔에서 이 예제를 실행해보면, 콘솔에서는 SIZE를 obj의 프로퍼티로 나열하지 않는 걸 볼 수 있음
obj[SIZE]를 입력해보면 SIZE가 obj의 프로퍼티인 것을 확인 할 수 있지만, 심볼 프로퍼티는 다르게 처리되며 기본적으로는 표시되지 않음

또한 이 프로퍼티의 키는 SIZE 심볼이며, 문자열 "SIZE"가 아님!

obj.SIZE = 0;을 입력할 후

- obj[SIZE]
- obj.SIZE or obj["SIZE"]
를 입력해보면 확인할 수 있음

중괄호 안에서
각 프로퍼티는 쉼표로 구분,
프로퍼티 이름과 값은 콜론으로 구분

1. Number.INFINITY와 같은 특별한 값을 저장

ex)
const s = 'hello';

위 예제에서 s는 원시타입이나 객체처럼 toUpperCase()라는 함수 프로퍼티를 가진 것 처럼 보임.
how?
JS는 일시적인 String 객체를 만듦
그 임시 객체에 toUpperCase 함수가 들어있음
=> JS는 함수를 호출하는 즉시 임시 객체를 파괴


ex)
const s = 'hello';
s.rating = 3; //에러는 없음. 과연 3값은 성공적으로 저장되었을까?
s.rating; // Undefined

위 예제는 마치 문자열 s에 프로퍼티를 할당하는 것 처럼 보임
사실은 일시적은 String 객체에서 프로퍼티를 할당한 것임.
그리고 임시 객체는 즉시 제거되므로 s.rating은 undefined 임

	- 배열 콘텐츠에는 항상 순서가 있음
	- 키는 순차적인 숫자

JS 배열은
	- C 언어의 효율적인 배열(indexed array)와 
	- 더 강력한 동적 배열, 링크드 리스트(linked list)

배열의 특징
- 배열 크기는 고정되지 않음. 언제든 요소를 추가하거나 제거할 수 있음
- 요소의 데이터 타입을 가리지 않음.(즉, 문자열만 쓸 수 있는 배열 그런건 없음)
- 배열 요소는 0으로 시작(배열도 객체의 일종이므로, 숫자가 아닌 키나 분수, 음수를 사용할 수는 있음. 하지만 이런 시도는 찾기 어려운 버그를 야기할 수 있으므로 하지 말아야함)


배열에는 요소 숫자를 반환하는 length 프로퍼티가 있음
